## 1. Predicates - 술어 

### 정리

1. 술어(Predicates)
    - 명제는 완성된 문장으로, 해당 문장만으로 참 / 거짓을 판별할 수 있다.
    - 하지만 술어는 아직 완성되지는 않은 문장으로, 해당 문장의 변수에 따라 참 / 거짓이 달라질 수 있는 문장이다.
    - 즉, 술어란 변수의 값이 정해지지 않은 열린 문장으로,
    변수에 어떤 값이 들어가느냐에 따라 참 / 거짓이 달라질 수 있다.
    - 예 ) 
      - $z = x + y$ ($z$, $x$, $y$가 확정되어야 참 / 거짓을 판단할 수 있다.)
      - $x > 3$ ($x$가 확정되어야 참 / 거짓을 판단할 수 있다.)
      - $length가\;4096이하이다.$ ($length$가 확정되어야 참 / 거짓을 판단할 수 있다.)
        ```cpp
        if (length <= 4096) {
        	// 술어 length <= 4096이 참일 때만 실행
            // 변수 length에 값이 들어간다면 명제로 바뀌어서 참 / 거짓을 판단할 수 있음
        }
        ```
2. 명제함수(Propositional function)
    - 술어 $x + 1 = 2$라고 할 때,
    이 술어를 명제함수 $P(x) : x + 1 = 2$로 표현할 수 있다.
    - 이 함수의 치역(반환값)은 참($T$) / 거짓($F$)이다.
    말 그대로 해당 술어에 대한 함수이다.
    - 명제함수는 어떤 값을 입력받아 참($T$) 또는 거짓($F$)을 반환하는 함수처럼 볼 수 있다.
    - 이 명제함수에 특정 값을 대입하게 된다면,
    술어는 명제가 된다.
    - 예 )
      - $P(x) : x + 1 = 2$
      $P(3) : 3 + 1 = 2$(명제)
      $P(3) = F$ (명제이므로 참 / 거짓을 판단 가능)
    - 물론, 여러개의 변수를 포함하는 술어도 있는데, 그런 술어들도 명제함수로 표현할 수 있다.
      - 예 ) 
        - $P(z, x, y) : z = x + y$
        $P(3, 2, 1) : 3 = 2 + 1$ (명제)
        $P(3, 2, 1) = T$ (명제이므로 참 / 거짓을 판단 가능)
      - $P(x_{1}, x_{2}, ... , x_{n})$처럼 여러 개의 변수를 포함하는 술어를
      n-변수 술어(n-place predicate) / n-항 술어(n-ary predicate)라고 한다.
3. 전조건과 후조건
    - 술어는 프로그래밍할 때도 올바른 입력이 들어왔을 때
    항상 원하는 출력을 산출한다는 것을 확증할 때도 사용된다.
    - 이 때, 
    코드나 함수가 실행되기 전에 만족되어야 하는 조건 = 전조건
    코드나 함수가 실행된 후 만족되어야 하는 결과 또는 상태 조건 = 후조건
    이라고 할 수 있다.
    - 예 ) 
    백준 문제에서는 입력 제한이 전조건에 가깝고,
    문제에서 요구하는 정답 형식과 출력 조건이 후조건에 가깝다.

## 2. Quantifiers - 한정기호

### 정리

1. 한정기호(Quantifiers)란?
    - 술어를 표현하는 명제함수는 변수에 값이 대입되어야 참 / 거짓을 판별할 수 있는 명제가 된다.
    - 한정기호란, 명제함수가 정의역(넣을 수 있는 변수값들)의 범위 내에서 
    어떤 방식으로 참 / 거짓을 판단받는지 정해주는 기호이다.
    - 즉, 명제함수의 변수들의 값을 하나로 정하는 것이 아니라,
    명제함수의 정의역의 범위 내의 값들로 나온 
    치역의 값들(각 정의역의 값에 대해 나온 참 / 거짓 결과들)을
    어떻게 최종적으로 하나의 참 / 거짓으로 합치는지에 대한 규칙이다.
    - 예 )
        - $\forall x P(x)$ 
        = 명제함수 $P(x)$에서 
        정의역의 모든 $x$에 대해 모든 $P(x)$(치역)가 참일 때 최종적으로 참임
          - 즉, 명제로 표현하면
          > 모든 $x$에 대하여 명제함수 $P(x)$가 참인가?
        - $\exists x P(x)$ 
        = 명제함수 $P(x)$에서 
        정의역의 모든 $x$에 대해 $P(x)$(치역) 중에 
        하나라도 참으로 만드는 $x$가 있으면 최종적으로 참임
          - 즉, 명제로 표현하면
          > 명제함수 $P(x)$의 모든 $x$ 중에 하나라도 참인 경우가 있는가?
    - 이렇게 명제함수의 정의역의 범위 내의 값들로 나온 치역의 값들을
    최종적으로 하나의 진리값으로 합쳐서 명제함수를 명제로 만들어주는 표현을 한정기호라고 한다.
    - 한정기호 $\forall$과 $\exists$는 모든 논리 연산자보다 상위의 우선순위를 가진다.
2. 전체 한정기호(universal quantifier)
     - $\forall x P(x)$
     > 모든 $x$에 대하여 $P(x) = T$이다.
     - 정의역 내의 모든 $x$값에 대해서 $P(x) = T$여야만 최종적으로 참($T$)이 된다는 의미의 한정 기호이다.
     - 정의역 내에 단 하나라도 $P(x) = F$인 $x$값이 존재한다면, 최종적으로 해당 명제의 진리값은 거짓($F$)이 된다.
     - $\forall x P(x)$(전체 한정기호)의 반례 : ($P(x) = F$)가 되는 정의역의 원소($x$)
     반례를 찾으면 전체 한정기호($\forall$)로 만들어진 명제의 진리값이 거짓($F$)인 것을 알 수 있다.
     - 프로그래밍과의 관련성 : `AND` 연산($\land$)의 확장판.
       - $P(x_1) \land P(x_2) \land \dots \land P(x_n)$ 
       ($n$은 정의역의 원소의 개수)
       (정의역의 원소가 유한한 경우)
       - 파이썬 예시 )
       (한 배열의 모든 원소를 순회한다는 의미를 표현하기에 적합해서 파이썬을 사용)
       ```python
       for n in bool_arr:
           if (n == False):
               return False
        return True
       ```
       - C++에서는 `algorithm` 헤더의 `std::all_of()` 함수가 전체 한정기호를 구현한다.
     - 부정($\neg \forall$) : $\neg \forall x P(x) \equiv \exists x \neg P(x)$
       - $\neg \forall x P(x)$는 
       > 정의역 내의 모든 $x$가 $P(x) = T$를 만족하는 것은 아니다.
       
       - 라는 의미이다.
       즉, 
       > 정의역 내의 모든 $x$ 중에서 $P(x) = F$인 경우가 존재한다.
       
       - 라는 의미라고 할 수 있다.
       - $\exists x P(x)$는 정의역 내의 모든 $x$ 중에 하나라도 $P(x) = T$를 만족한다는,
       즉
       > 정의역 내의 모든 $x$ 중에서 $P(x) = T$인 경우가 존재한다.
       
       - 라는 의미이다.
       - 만약 $\exists xP(x)$에서 $P(x)$를 부정한다면($\exists x \neg P(x)$),
       > 정의역 내의 모든 $x$ 중에서 $P(x) = F$인 경우가 존재한다.
       
       - 는 의미가 된다.
       - 그러므로 $\neg \forall x P(x) \equiv \exists x \neg P(x)$이다.
     - EX. 정말 쉽게 $\neg \forall x P(x)$ 이해하기
       - $\neg \forall x P(x) \equiv \neg (P(x_1) \land P(x_2) \land \dots \land P(x_n))$
       (드 모르간의 법칙 적용)
       - $\equiv (\neg P(x_1) \lor \neg P(x_2) \lor \dots \lor \neg P(x_n))$
3. 존재 한정기호(existential quantifier)
     - $\exists x P(x)$
     > 모든 $x$ 중에서 $P(x) = T$인 경우가 (하나라도) 존재한다.
     - 정의역 내의 모든 $x$값 중에서 $P(x) = T$인 원소($x$)가 하나라도 존재하면 최종적으로  참($T$)이 된다는 의미의 한정기호이다.
     - 정의역 내의 모든 $x$값에 대해서 $P(x) = F$이라면, 최종적으로 해당 명제의 진리값은 거짓($F$)이 된다.
     - 프로그래밍과의 관련성 : `OR` 연산($\lor$)의 확장판.
       - $P(x_1) \lor P(x_2) \lor \dots \lor P(x_n)$ 
       ($n$은 정의역의 원소의 개수)
       (정의역의 원소가 유한한 경우)
       - 파이썬 예시 ) 
       ```python
       for n in bool_arr:
           if (n == True):
               return True
        return False
       ```
       - C++에서는 `algorithm` 헤더의 `std::any_of()` 함수가 존재 한정기호를 구현한다.
       - 부정($\neg \exists$) : $\neg \exists x P(x) \equiv \forall x \neg P(x)$
       - $\neg \exists x P(x)$는
       > 정의역 내의 $x$ 중에서 $P(x) = T$인 경우가 존재하는 것은 아니다.
       
       - 라는 의미이다.
       즉, 
       > 정의역 내의 $x$중에서 $P(x) = T$인 경우가 존재하지 않는다.
       
       - 라는 의미라고 할 수 있다.
       - $\forall x P(x)$는 정의역 내의 모든 $x$가 $P(x) = T$를 만족한다는 것이다.
       즉,
       > 정의역 내의 모든 $x$에 대해서 $P(x) = T$이다.
       
       - 라는 의미이다.
       - 만약 $\forall x P(x)$에서 $P(x)$를 부정한다면($\neg P(x)$),
       > 정의역 내의 모든 $x$에 대해서 $P(x) = F$이다.
       > 정의역 내의 모든 $x$에 대해서 $P(x) = T$인 경우가 존재하지 않는다.
       
       - 는 의미가 된다.
       - 그러므로 $\neg \exists x P(x) \equiv \forall x \neg P(x)$이다.
     - EX. 정말 쉽게 $\neg \exists x P(x)$ 이해하기
       - $\neg \exists x P(x) \equiv \neg (P(x_1) \lor P(x_2) \lor \dots \lor P(x_n))$
       (드 모르간의 법칙 적용)
       - $\equiv (\neg P(x_1) \land \neg P(x_2) \land \dots \land \neg P(x_n))$
4. 유일 한정기호
     - $\exists_1 x P(x)$
     > 모든 $x$ 중에서 하나의 $x$만이 $P(x) = T$이다.
     - 정의역 내의 모든 $x$값 중에서 $P(x) = T$인 원소($x$)가 단 하나이면 최종적으로 참 ($T$)이 된다는 의미의 한정기호이다.
     - 정의역 내의 모든 $x$값에 대해서 $P(x) = F$거나 $P(x) = T$가 두 개 이상이라면,
     최종적으로 해당 명제의 진리값은 거짓($F$)이 된다.
     - 유일성은 $\exists!$로 쓰기도 하고, 
     전체/존재 한정기호와 등호를 이용해 풀어 쓸 수도 있다
     
5. 한정기호의 동치
    - 한정기호가 포함된 명제에서
      - 논리식의 형태는 다르지만,
      - 즉, 술어(명제함수)가 달라졌어도
      - 같은 진리값을 가지는 명제를 한정기호의 논리적 동치라고 한다.
      - 예 )
        - $\forall x (P(x) \land Q(x)) \equiv \forall xP(x) \land \forall xQ(x)$
        두 명제는 논리적 동치이다.
        - 물론 
          - 전체 한정 = 논리곱(`AND`, $\land$)의 집합
          - 존재 한정 = 논리합(`OR`, $\lor$)의 집합
        - 이므로 전체 한정은 $\land$만,
        존재 한정은 $\lor$만 분배법칙을 사용한 논리적 동치가 성립한다.
      
     
     
6. 요약
 - | 한정기호      | 유한 정의역에서의 느낌 | C++ 느낌   | 부정 |
     | --------- | ------------ | ---------- | ------- |
     | $\forall$ ($\forall x P(x)$) | 거대한 AND      | `all_of()` | $\exists x \neg P(x)$ |
     | $\exists$ ($\exists x P(x)$) | 거대한 OR       | `any_of()` | $\forall x \neg P(x)$ |

1. 일단 한정기호의 정의를 직관적으로 이해하는게 너무 어렵다..
내가 직관적으로 이해한 한정기호의 정의는,
> 특정 명제함수의 정의역 $x$에 대해 명제함수에서 나온 결과인 치역($T$ / $F$)들을
최종적으로는 어떻게 하나의 진리값($T$ / $F$)로 합치는 지에 대한 규칙
2. 한정기호를 부정하는게 어려워도 너무 어렵다..
내가 직관적으로 이해한 한정기호의 부정은
> 한정기호($\forall$ / $\exists$)의 의미 자체를 부정하는 느낌.
즉, 반례.
$\forall$ = 정의역의 모든 $x$에 대해서 $P(x) = T$인 경우만 있다.
$\neg \forall$ = 정의역의 모든 $x$에 대해서 $P(x) = T$이 아닌 경우가 있다. 
($\forall$의 의미 자체를 부정, 즉 반례)
$\exists$ = 정의역의 모든 $x$ 중에 $P(x) = T$인 경우가 하나라도 있다.
$\neg \exists$ = 정의역의 모든 $x$ 중에서 $P(x) = T$인 경우가 하나도 없다.
($\exists$의 의미 자체를 부정, 즉 반례)

그냥 논리적인 직관을 100% 끌어다써야하는 개념들이다보니까..

결국 이런 것들은 어떻게 쓰이는지 생각해봤는데,

1. 버그 가능성 탐색
    - 이전 글에서 쓴 버그 가능성을 확인할 수 있는 명제인 
    $BugPossible = ReachablePath \land BadState$도
    이것도 사실 술어와 한정기호로 만들어진 명제인 것 같음.
    - 술어(명제함수)와 한정기호($\exists$), 그리고 논리 연산자($\land$)로 수정해보면, 
    $BugPossible = \exists a(ReachablePath(a) \land BadState(a))$
    - > 버그가 발생할 수 있다는 것은, 
    해당 라인까지 정상적으로 도달했다는 가정 하에($ReachablePath(a) = T$)
    버그가 발생할 수 있는 상태인 경우($BadState(a) = T$)가
    단 하나라도 있을 때($\exists$)임.
    - 이렇게 되는 것 같아.
    - 물론 해당 라인까지 도달했을 때의 조건은 
    $a$ 하나뿐이 아니겠지.
2. 한정기호가 포함된 명제의 반례 찾기
    - 한정기호($\forall$ / $\exists$)가 포함된 명제의 반례는
    해당 한정기호를 부정함으로써 찾을 수 있다.
    
이 정도가 떠오르네.

## 총 요약

이 글을 총 요약해보자면,

> 명제 = 문장 자체로 참/거짓 판단 가능

> 술어 = 변수 값에 따라 참/거짓이 달라지는 문장

> 명제함수 = 술어를 함수처럼 표현한 것
$P(x)$처럼 쓰고, 명제함수의 매개변수에 값을 넣으면 명제가 됨

> 전체 한정기호 $\forall$ = 정의역($x$)에 의한 명제함수의 결과값(치역)이 전부 참이어야 참
거대한 `AND` 
$P(x_1) \land P(x_2) \land \dots \land P(x_n)$
`C++` : `all_of()`

> 존재 한정기호 $\exists$ = 정의역($x$)에 의한 명제함수의 결과값(치역)이 하나라도 참이면 참
거대한 `OR` 
$P(x_1) \lor P(x_2) \lor \dots \lor P(x_n)$
`C++` : `any_of()`

> $\neg \forall x P(x)$ 
= 모든 $x$에서 $P(x) = T$인 것은 아니다
= $P(x) = T$가 아닌 $x$가 존재한다
= $P(x) = F$인 $x$가 존재한다
= $\exists x \neg P(x)$

> $\neg \exists x P(x)$ 
= $P(x) = T$인 $x$가 존재하지 않는다
= 모든 $x$가 $P(x) = T$가 아니다
= 모든 $x$가 $P(x) = F$이다
= $\forall x \neg P(x)$
