## 0. 서론

저번에는 단순히 한정기호 하나의 뜻과,
한정기호 하나의 부정을 공부했다면,

이번에는 한정기호 여러개가 한 명제 함수에 붙어있는 경우에 대한 것을 공부해봤어.

- $\forall x \exists y P(x, y)$
- $\exists x \forall y P(x, y)$

이딴것들.

이게 뭔가 어려워보이는데,
단순히
- 구조적으로 이해할 때는 안쪽(오른쪽)에서부터 의미를 해석하는게 편하다.
- 자연어로 표현할 때는 왼쪽부터 의미를 해석하는게 편하다.
- 어차피 한정기호로 만들어진 명제에 한정기호를 더 씌울 뿐이다.

이 세 가지만 알고있으면 그리 어렵지는 않은 것 같아.

어쨌든
이번에는,
**1.5 - 중첩된 한정기호**
에 대해 공부했어.

## 1. 중첩된 한정기호

### 정리

1. 한정기호를 중첩해도 한정기호로 만들어진 명제에 추가로 한정기호를 씌우는 것과 다르지 않다.
    - $\forall x \exists y P(x, y) = \forall x (\exists y P(x, y))$
    $\exists y P(x, y) = Q(x)$라고 하면
    $\forall x Q(x)$와 같다고 할 수 있다.
    - 뜻은,
    $Q(x) = \exists y P(x, y)$
    > 정의역의 모든 $y$ 중에서 $P(x, y) = T$인 경우가 존재한다.
    
    - $\forall x Q(x)$
    > 정의역의 모든 $x$에 대해서 $Q(x) = T$인 경우만 존재한다.
    
    - 즉, 
    > 정의역의 모든 $x$에 대해서 정의역의 $y$ 중에서 $P(x, y) = T$인 경우가 존재한다..
    
    - 좀 더 쉽게 풀면,
    > 정의역의 모든 $x$에 대해서, $P(x, y) = T$를 만족시키는 정의역의 $y$가 존재한다.
    > 모든 $x$마다, $P(x, y) = T$를 만족시키는 $y$가 적어도 하나 존재한다.
    
    - 라고 할 수 있다.
    - $\exists y \forall x P(x, y) = \exists y (\forall x P(x, y))$
    $\forall x P(x, y) = Q(y)$라고 하면
    $\exists y Q(y)$와 같다고 할 수 있다.
    - 물론 뜻은,
    $Q(y) = \forall x P(x, y)$
    > 정의역의 모든 $x$에 대해서 $P(x, y) = T$인 경우만 존재한다.
    
    - $\exists y Q(y)$
    > 정의역의 모든 $y$ 중에 $Q(y) = T$인 경우가 존재한다.
    
    - 즉,
    > 정의역의 모든 $y$ 중에 정의역의 모든 $x$에 대해서 $P(x, y) = T$인 경우만 존재한다..
    
    - 좀 더 쉽게 풀면,
    > 정의역의 모든 $y$ 중에 모든 $x$가 $P(x, y) = T$를 만족시키는 경우가 존재한다.
    > 모든 $y$ 중 하나라도 모든 $x$에 대해서 $P(x, y) = T$를 만족시키는 $y$가 존재한다.
    
    - 라고 할 수 있다.
    
2. 쉬운 이해 - 다중 반복문
    - $\forall x \exists y P(x, y)$는 반복문으로 보면 다음과 같은 구조이다.
	```cpp
	for (int x = 0; x < range; x++) { // 모든 x에 대해서
    	bool exists_y = false;

    	for (int y = 0; y < range; y++) { // y 중에 하나라도 P(x, y) = T인 경우가 있다면
        	if (P(x, y) == true) {
            	exists_y = true;
            	break;
        	}
    	}

    	if (exists_y == false) {
        	return false;
    	}
	}

	return true;
	```
	- 바깥 반복문은 모든 $x$를 검사하는 $\forall x$에 해당하고,
	안쪽 반복문은 현재 $x$에 대해 $P(x, y)$를 참으로 만드는 $y$가 하나라도 존재하는지 검사하는 $\exists y$에 해당한다.
	- 즉, 모든 $x$가 각각 $P(x, y) = T$를 하나라도 달성할 수 있다면 참이다.
    - $\exists y \forall x P(x, y)$는 반복문으로 보면 다음과 같은 구조이다.
    ```cpp
    bool is_y_passed = false;
    
    for (int y = 0; y < y_range; y++) { // y 중에 하나라도
    
    	bool all_x_passed = true;
    
    	for (int x = 0; x < x_range; x++) { // 모든 x에 대해 P(x, y) = T인 경우만 있다면
            if (P(x, y) == false) {
                all_x_passed = false;
                break;
            }
            
        }
        
        if (all_x_passed == true) return true; 
    
    }
    
    return false;
    ```
    - 바깥 반복문은 '하나의 $y$라도'를 의미하는 $\exists y$에 해당하고,
    안쪽 반복문은 해당 $y$가 모든 $x$에 대해 $P(x, y)$를 참으로 만드는지 검사하는 $\forall x$에 해당한다.
    - 즉, 하나의 $y$라도 모든 $x$에 대해서 $P(x, y) = T$가 달성된다면 참이다.
    
3. 다른 한정기호를 중첩한다면 한정기호의 순서가 상당히 중요해진다.
    - 앞에서처럼, 
    $\forall x \exists y P(x, y)$와 $\exists y \forall x P(x, y)$는
    의미가 다르다.
    > 모든 $x$에 대해서 $y$중에 $P(x, y) = T$인 경우가 하나라도 있나? ($\forall x \exists y P(x, y)$)
    vs $y$중에서 모든 $x$가 $P(x, y) = T$인 경우가 하나라도 있나? ($\exists y \forall x P(x, y)$)
    
    - 그렇기 때문에 다른 한정기호를 중첩한다면 한정기호의 순서가 상당히 중요하다.
    
4. 같은 한정기호가 중첩된다면, 기존의 한정기호와 판단 방식은 그대로이고 판단 대상만 늘어난다.
    - 예를 들어, $\forall x P(x)$는
    > 정의역의 모든 $x$에 대해서 $P(x) = T$인 경우만 존재한다.
    
    - 라는 의미이다.
    - 하지만, $\forall x \forall y P(x, y)$처럼 같은 한정기호를 중첩해도,
    > 정의역의 모든 $x$와 $y$에 대해서 $P(x, y) = T$인 경우만 존재한다.
    
    - 라는 의미가 된다.
    - 즉, 한정기호로 만들어진 명제의 진리값 판단 대상이
    $x$ 하나에서 $(x, y)$의 조합으로 늘어났을 뿐,
    핵심은 "모든 경우에 참이어야 참"이라는 것이다.
    - 그래서 유한한 정의역에서는 이렇게 생각할 수 있다.
    $\forall x \forall y P(x, y)$
    $= P(x_1, y_1) \land P(x_1, y_2) \land \dots \land P(x_2, y_1) \land P(x_2, y_2) \land \dots \land P(x_n, y_n)$
    이렇게 단순히 기존의 전체 한정기호($\forall$)에서 판단 대상만 늘어난 것이라고 생각할 수 있다.
    - 물론 존재 한정기호($\exists$)도 마찬가지다.
    - $\exists x P(x)$는
    > 정의역의 $x$ 중에 $P(x) = T$인 경우가 존재한다.
    
    - 라는 의미이다.
    - 하지만, $\exists x \exists y P(x, y)$처럼 같은 한정기호를 중첩해도,
    > 정의역의 $x$와 $y$의 쌍 중에 $P(x, y) = T$인 경우가 존재한다.
    
    - 라는 의미가 된다.
    - 즉, 여기서도 한정기호로 만들어진 명제의 진리값 판단 대상이
    $x$ 하나에서 $(x, y)$의 조합으로 늘어났을 뿐,
    핵심은 "모든 경우 중 하나로 참이라면 참"이라는 것이다.
    - 그래서 유한한 정의역에서는 이렇게 생각할 수 있다.
    $\exist x \exists y P(x, y)$
    $= P(x_1, y_1) \lor P(x_1, y_2) \lor \dots \lor P(x_2, y_1) \lor P(x_2, y_2) \lor \dots \lor P(x_n, y_n)$
    이것도 단순히 기존의 존재 한정기호($\exists$)에서 판단 대상만 늘어난 것이라고 생각할 수 있다.
    
5. 같은 한정기호를 중첩한다면 순서를 바꿔도 의미가 달라지지 않는다.
    - $\forall x \forall y P(x, y)$
    > 모든 $x$에 대한 모든 $y$가 $P(x, y) = T$인 경우만 존재한다.
    > = 모든 $x$와 $y$에 대해서 $P(x, y) = T$인 경우만 존재한다.
    > = 모든 $x$와 $y$의 쌍이 $P(x, y) = T$를 만족한다.
    
    - $\forall y \forall x P(x, y)$
    > 모든 $y$에 대한 모든 $x$가 $P(x, y) = T$인 경우만 존재한다.
    > = 모든 $y$와 $x$에 대해서 $P(x, y) = T$인 경우만 존재한다.
    > = 모든 $y$와 $x$의 쌍이 $P(x, y) = T$를 만족한다.
    
    - 결론은 $\forall$를 중첩한다면 의미가 크게 변하지 않는다.
    $\forall x \forall y P(x, y) \equiv \forall y \forall x P(x, y)$
    어쨌든 둘 다 
    > 모든 $(x, y)$ 쌍에서 $P(x, y) = T$인 경우만 존재한다.
    
    - 라는 의미이기 때문이다.
    - $\exists x \exists y P(x, y)$
    > $x$와 $y$의 쌍 중에 $P(x, y) = T$인 경우가 존재한다.
    
    - $\exists y \exists x P(x, y)$
    > $y$와 $x$의 쌍 중에 $P(x, y) = T$인 경우가 존재한다.
   	
    - 즉, $\exists$를 중첩해도 의미가 크게 변하지 않는다.
    $\exists x \exists y P(x, y) \equiv \exists y \exists x P(x, y)$
    어쨌단 둘 다 
    > 모든 $(x, y)$ 쌍에서 $P(x, y) = T$인 경우가 존재한다
    
    - 라는 의미이기 때문이다.

## 2. 중첩된 한정기호의 부정

### 정리

1. 한정기호 상으로는 $\neg \forall \equiv \exists \neg$, $\neg \exists \equiv \forall \neg$임을 이용하면 
쉽게 중첩된 한정기호의 부정을 표현할 수 있다.
   - 예 ) $\neg \forall x \exists y P(x, y)$
   $\equiv \exists x \neg \exists y P(x, y)$
   $\equiv \exists x \forall y \neg P(x, y)$
   - 예 ) $\neg \exists y \forall x P(x, y)$
   $\equiv \forall y \neg \forall x P(x, y)$
   $\equiv \forall y \exists x \neg P(x, y)$
   
2. 자연어 상으로도 기존의 한정기호의 부정 규칙을 반복해서 적용하면 된다.
   - $\forall x \exists y P(x, y)$
   > 정의역의 모든 $x$에 대해서 정의역의 $y$ 중에서 $P(x, y) = T$를 만족하는 경우가 존재한다.
   > 모든 $x$가 $P(x, y) = T$인 $y$를 하나 이상 가진다.
   
   - 부정하면($\neg \forall x \exists y P(x, y)$),
   > 정의역의 $x$ 중에 하나라도 정의역의 $y$ 중에서 $P(x, y) = T$를 만족하지 못하는 경우가 존재한다. 
   > 정의역의 $x$ 중에 하나라도 정의역의 모든 $y$가 $P(x, y) = F$를 만족하는 경우가 존재한다.
   
   - 라는 의미가 된다.
   - 즉, $x$ 중에서 적어도 하나에서 어떤 $y$여도 $P(x, y) = F$를 만족하는 경우가 존재한다는 의미이다.
   - 이걸 한정기호로 표현한다면 $\exists x \forall y \neg P(x, y)$가 된다.
   - $\exists y \forall x P(x, y)$
   > 정의역의 $y$ 중에서 모든 $x$에 대해 $P(x, y) = T$를 만족하는 경우가 존재한다.
   > 모든 $y$ 중 하나 이상이 모든 $x$에서 $P(x, y) = T$인 경우를 가진다.
   
   - 부정하면($\neg \exists y \forall x P(x, y)$)
   > 모든 정의역의 $y$에 대해서, 각각의 $y$에서 정의역의 $x$들이 전부 $P(x, y) = T$를 만족하는 경우만 존재하지 않는다.
   > 모든 정의역의 $y$에 대해서, 각각의 $y$에서 정의역의 $x$ 중에서 하나라도 $P(x, y) = F$를 만족하는 경우가 존재한다. 
   
   - 라는 의미가 된다.
   - 모든 $y$에서, 각각의 $y$일 때의 모든 $x$가 $P(x, y) = F$를 만족시키는 경우가 존재한다는 의미이다.
   - 이걸 한정기호로 표현한다면 $\forall y \exists x \neg P(x, y)$가 된다.
