## 1. Fractional Binary Numbers - 2진수 소수

### 정리

1. 이진수 소수는 정수 부분과 소수 부분을 나눠서 표현된다.
    - 10진수 소수는 $n.m$처럼, 정수 부분과 소수 부분이 나뉘어 표현된다.
    - 2진수 소수도 $n.m$처럼, 정수 부분과 소수 부분이 나뉘어 표현된다.
    - 10진수 소수는 각 자리의 가중치가 $10^i$라고 할 수 있고,
    정수 부분은 $i \geq 0$의 범위, 즉 $10^0$(일의 자리), $10^1$(십의 자리), $10^2$(백의 자리)...로 표현된다.
    소수 부분은 $i < 0$의 범위, 즉 $10^{-1}$($\frac{1}{10}$), $10^{-2}$($\frac{1}{100}$)..로 표현된다.
    - 수식화하면 이런 느낌이다.
    $$
    d = \sum_{{i = -n}}^{m} 10^{i} \times d_{i}
    $$
    - 2진수 소수도 물론 각 자리의 가중치라 $2^i$라고 할 수 있고,
    정수 부분은 $i \geq 0$의 범위, 즉 $2^{0}$, $2^{1}$, $2^{2}$...로 표현된다.
    소수 부분은 $i < 0$의 범위, 즉 $2^{-1}$($\frac{1}{2}$), $2^{-2}$($\frac{1}{4}$), $2^{-3}$($\frac{1}{8}$)...로 표현된다.
    - 수식화하면 이런 느낌이다.
    $$
    b = \sum_{i = -n}^{m} 2^{i} \times b_{i}
    $$
    - 즉, 2진수 소수도 10진수 소수와 동일한 구조라고 볼 수 있다.
    - 다만 각 자릿수의 가중치가 $10^{i}$가 아닌 $2^{i}$일 뿐이다.
    - 2진수 소수 예시 :
      - $0.25_{10} = 0.01_{2}$ ($0.25_{10} = \frac{1}{4} = 2^{-2} = 0.01_{2}$)
      - $0.1875_{10} = 0.0011_{2}$ ($0.1875_{10} = \frac{3}{16} = \frac{1}{8} + \frac{1}{16} = 2^{-3} + 2^{-4} = 0.0011_{2}$)
      - $0.101_{2} = 0.625_{10}$ ($0.101_{2} = 2^{-1} + 2^{-3} = \frac{1}{2} + \frac{1}{8} = \frac{5}{8} = 0.625_{10}$)
      - $0.00110011_{2} = 0.19921875_{10}$ 
      ($0.00110011_{2} = 2^{-3} + 2^{-4} + 2^{-7} + 2^{-8} = \frac{1}{8} + \frac{1}{16} + \frac{1}{128} + \frac{1}{256} = \frac{51}{256} = 0.19921875_{10}$)  
      - $101.11_{2} = 5.75_{10}$ ($101.11_{2} = 2^{2} + 2^{0} + 2^{-1} + 2^{-2} = 4 + 1 + \frac{1}{2} + \frac{1}{4} = 5\frac{3}{4} = 5.75_{10}$)
2. 무한 소수 문제
    - 10진수 소수에도 $\frac{1}{3}$같은 숫자는 $0.33333..._{10}$으로 무한 소수이다.
    - 물론 2진수 소수에도 무한 소수가 발생할 수 있다.
      - $0.1_{10} = \frac{1}{10} = 0.0001100110011..._{2}$같은 수가 해당된다.
    - 2진수 소수에서는 유한한 비트들로 표현할 수 있는 수는
    정수 $x$와 정수 $y$에 대해 $x \times 2^{y}$ 형태의 수 들 뿐이다.
    - 그래서, 2진수 소수에서는 유한한 비트들로 표현할 수 없는 수는
    근사치로 표현된다.
    
### 느낀 점 & 배운 점

그냥 어떻게 2진수로 소수를 표현할 수 있고,
그렇게 표현된 소수를 어떻게 해석할 수 있는지 알려주는 파트인 것 같았어.

결국
> 각 자리의 가중치가 $2^{k}$일 뿐이고,
소수점도 결국 $2^{-k}$의 가중치들로 표현될 뿐이다.

근데 여기서 중요한 점이,
지수가 음수일 때 소수점이 표현된다는거지.

$2^{-1} = \frac{1}{2}$
이렇게.

사실 이미 어느정도는 알고 있었는데,
$x \times 2^{y}$ 형태로만 정확한 값을 표현할 수 있다는건 처음 알았어.

이게 결국, 
10진수 소수에서 분모가 $10^k$가 될 수 없다면(예 : 분모가 3, 7 등) 무한소수가 되는 것처럼,
2진수 소수에서는 분모가 $2^k$가 될 수 없다면(예 : 분모가 3, 5, 6 등)
결국 무한소수가 되어서 한정된 $w$비트 안에서는 결국 근사치로 표현된다는게 신기했어.

근데 결국 10진수 표현으로는 무한소수가 아닌 소수가,
2진수 표현으로는 무한소수가 될 수 있다는게 신기했어.
이게 뭔가 핵심인 것 같네.

위에서 봤듯이 $0.1_{10}$이 10진수에서는 무한소수가 아니지만 
2진수에서는 $0.0001100110011...$이 반복되는 무한소수인 것처럼.

사실 예전에 코딩애플 유튜브 보고 대충 2진수 무한소수는
실제 기대하는 값의 근사치로 표현된다는건 알고 있었는데,
$x \times 2^{k}$라는 수식은 이번에 처음 본 것 같아.

그 영상에서 설명한 것도 무한소수는 한정된 $w$비트 안에 담기니
완벽히 표현할 수 없어서 근사치로 표현되고,
그래서
```cpp
0.1 + 0.2 != 0.3
```
이렇게 된다는 내용이었거든.

$0.1$은 위에서 2진수 소수로 봤을 때는 무한소수란걸 보였고,
$0.2$도 $0.001100110011..._{2}$로 무한 소수이니
결국 둘 다 정확한 값이 아닌 근사치로 표현되어서
위의 식이 성립한다는 거였어.

그래서 그 영상의 결론이

- 애초에 그냥 중요한 값들, 오차가 발생하면 안되는 값들은
소수로 표현하지 말고 정수로 표현하자.
- 굳이 소수로 표현해야겠으면 `double` 형같은 정밀도가 더 높은 자료형을 사용하자.

이랬거든.

이제 다음 챕터들에는 본격적으로 컴퓨터가 어떻게 소수를 표현하는지에 대한 포맷이 나올 것 같네.

## 2. IEEE Floating-Point Representation -IEEE 부동소수점 표현

### 정리

1. IEEE 754는 컴퓨터에서 실수를 표현하는 국제 표준이다.
    - 과거에는 프로세서(CPU)마다 다른 실수 표현 방식을 썼었지만,
    IEEE 754로 통일되었다.
    
2. 과학적 표기법
    - IEEE 754의 실수 표현 방식은
    과학적 표기법과 상당히 비슷하다.
    - 과학적 표기법이란,
    수를 $\pm M \times 10^{E}$처럼,
    숫자의 형태 따로, 숫자의 소수점 위치 따로 표현하는 방식이다.
    - 예를 들어, 
    $+3.14 \times 10^2 = 314$
    $-3.14 \times 10^{-2} = -0.0314$
    같은 느낌이다.
    - $\pm M \times 10^{E}$는 10진수 기반의 과학적 표기법이다.
    - 컴퓨터는 2진수 기반으로 실수를 표현하기 때문에,
    $\pm M \times 2^{E}$와 같이 과학적 표기법을 사용한다.
    - $+ 1.101_{2} \times 2^3 = 1101.0_{2}$
    $- 1.101_{2} \times 2^{-3} = -0.001101_{2}$
    같이 과학적 표기법이 적용된다.
    - 즉, $M$은 숫자의 생김새 자체를 나타내는 부분이고,
    $E$는 그 숫자의 소수점을 어디에 둘지 나타내는 지수이다.
    - 그리고 숫자의 모양을 표현하는 부분, $M$을 가수라고 하고,
    소수점 위치를 표현하는 부분, 즉 $2^E$의 $E$를 지수라고 한다.
    
3. IEEE 754의 부동소수점 표기 방식
    - IEEE 754에서는, 단순히 모든 비트를 각각의 가중치를 가지게 한다면
    소수를 표현하는데 너무나 많은 비트를 사용하게 되기 때문에,
    마치 과학적 표기법처럼
      - 부호를 나타내는 부호 비트 : $s$(sign bit)
      - 해당 수의 소수점을 얼마나 이동시킬지 나타내는 지수부(지수) : $exp$(exponent field)
      - 실제 수가 어떻게 생겼는지를 나타내는 가수부(유효 숫자) : $frac$(fraction field)
    - 로 세 부분으로 나뉘어서 실수를 나타낸다.
    - 수학적으로는, $V = (-1)^s \times {M} \times 2^{E}$,
    ($M$ = $frac$을 계산하여 나온 값, $E$ = $exp$를 계산하여 나온 값)
      - $M$은 $1 \sim 2 - \epsilon$ / $0 \sim 1 - \epsilon$ 범위를 가진다.
      - $E$는 2의 $E$제곱($E$는 음수일 수도 있음)을 곱해서 가중치를 부여해 
      소수점의 위치를 나타낸다.
    - 즉,
      - $s$가 부호를 정하고
      - $frac$이 표현할 수의 생김새를 나타내고
      - $exp$가 표현할 수의 어디에 소수점이 있는지를 나타낸다.
    - 하지만 저장된 필드인 $frac$과 $exp$가 바로 $M$과 $E$가 되는 것은 아니고,
    IEEE 754에 의한 별도의 계산 과정을 거치게 된다.
    
4. 정규화(Normalization)
    - 주어진 비트 표현이 나타내는 값은 $exp$에 따라 세 가지 경우로 표현된다.
    각각,
      - 정규화된 값 : $exp$가 일반적인 값(전부 1도 아니고 전부 0도 아님)
      - 비정규화된 값 : $exp$가 전부 0
      - 특수한 값 : $exp$가 전부 1
      
5. 정규화된 값(Normalized Values)
    - 정규화된 값은 $exp$의 모든 비트가 전부 0이거나 전부 1이 아닐 때,
    비트 패턴에 보정치(bias)를 넣어,
    지수가 음수인 경우도 포함하도록 $exp$를 해석하는 것이다.
    - 보정치는 $2^{k-1} - 1$에 해당하며,
    $exp$가 $8$비트인 `float` 형에서는 $2^{7} - 1 = 127$,
    즉 원래의 $8$비트 비트 패턴에서 $127$을 빼는 것으로 지수($exp$)가 음수일 수도 있도록
    보정하는 것이다.
    그렇게 보정이 된다면, $exp$의 범위는 모든 비트가 0이거나 1인 경우를 제외하면
    $-126 \sim +127$의 범위를 가지게 된다.
    - 즉, 정규화된 값에서 $exp$는 
    $E = exp - bias$의 형태로 $E$에 영향을 주게 된다.
    - 물론 $exp$의 크기가 큰 다른 자료형(`double` 등)에서는 보정치가 다르지만,
    $2^{k-1} - 1$이라는 보정치를 구하는 공식은 변하지 않는다.
    - 정규화된 값에서 $frac$은 
    위의 2진수 소수를 표현하는 방식처럼
    $0 \leq frac < 1$의 범위를 가지게 되며,
    ($0.f_{n-1}f_{n-2}\dots f_{0}$)
    최종적으로 그 값에 $1$을 더하게 되어 $M$이 만들어진다.
    ($M = 1 + frac$) ($M = 1.f_{n-1}f_{n-2}\dots f_{0}$)
    - 결국 $M$은 $1 \leq M < 2$의 범위를 가지게 된다.
    
6. 비정규화된 값(Denormalized Values)
    - 비정규화된 값은 $exp$의 모든 비트가 전부 $0$일 때의 해석 방식이다.
    - 정규화와 다른 점이 있다면,
    $E = exp - bias$가 아니라
    $E = 1 - bias$이 고정이라는 점이 있고,
    - $frac$에
    $1$을 더하지 않고,
    그대로 $M = frac$이라는 점이다.
    - 비정규화된 값은 두 가지 용도로 사용된다.
      - 1. 부동소수점에서 $0$을 표현할 수 있는 방법
        - 정규화된 값은 항상 $frac$에 1을 더하게 되기에
        $M$이 $1 \leq M < 2$가 되어
        $0$을 표현할 수 없다.
        - 만약 $M = f = 0$이라면,
        $0$을 표현할 수 있다.
        - 물론 $s$에 따라 $+0.0$과 $-0.0$이 둘 다 존재하겠지만,
        IEEE 754에서는 
        비트 표현은 달라도 일반적인 비교에서는 같은 값으로 취급된다.
        - 다만 일부 연산에서는 부호의 차이가 드러날 수 있다.
      - 2. $0.0$에 매우 가까운 수를 표현
        - 정규화된 값은 항상 $frac$에 1을 더하게 되기에
        $M$이 $1 \leq M < 2$가 되어
        $0$에 가까운 수를 표현하기 어렵다.
        - 그렇기에 $M = f = 0.f_{n-1}f_{n-2} \dots f_{0}$처럼
        $1$을 더하지 않는다면 $0$에 가까운 수도 표현할 수 있다.
        
7. 특수한 값들(Special Values)
    - 특수한 값은 $exp$의 모든 비트가 전부 $1$일 때,
    특수한 해석을 하는 것이다.
    - $frac$의 모든 비트가 전부 0이라면 infinity, 즉 무한대를 나타낸다.
      - $+\infty$ ($s = 0$)
      - $-\infty$ ($s = 1$)
    - $frac$의 모든 비트가 전부 0이 아니라면 $NaN$(Not a Number)을 나타낸다.
    - $NaN$은 정상적인 실수 값으로 표현할 수 없는 계산 결과를 나타내기 위해 사용된다.
      - $0 / 0$
      - $\sqrt {-n}$
      
8. 혹시 모를 의문점들
    - 왜 정규화된 값에서 $frac$에 $1$을 더할까?
      - 2진수에서 0이 아닌 수를 정규화하면 항상 $1.xxxx_{2}$꼴이 된다.
      - 왜냐하면 2진수는 0과 1로만 표현되고,
      0이 아닌 수의 첫 번째 유효 비트는 반드시 1이기 때문이다.
      - 예를 들어, $1101.0_2$는 $1.101_2 \times 2^3$처럼 정규화할 수 있고, 
      $0.001101_2$는 $1.101_2 \times 2^{-3}$처럼 정규화할 수 있다.
      - 이렇게 정규화를 하면 첫 번째 유효 비트는 반드시 1이 된다.
      - 그래서 IEEE 754에서는 이 앞의 1을 실제 비트로 저장하지 않고
      $frac$에는 그 뒤의 소수부만 저장하는 것이다.
      - 이후 값을 해석할 때는 생략했던 앞의 1을 다시 붙여서 $M$을 만드는 것이다.
    - 왜 비정규화된 값에서 $E = 1 - bias$일까?
      - 정규화된 값에서는 $exp$의 모든 비트가 전부 0일 수 없기 때문에,
      가장 작은 정규화된 $exp$는 $[000\dots 001]$이 된다.
      - 즉, $E = 1 - bias$는 정규화된 값의 가장 작은 실제 지수이다.
      - 비정규화된 값은 $0 \leq M < 1$ 범위의 수를 표현하기 위해 존재하는데,
      이 때 실제 지수 $E$를 정규화된 값의 가장 작은 지수와 동일하게 맞춰야
      정규화된 값과 비정규화된 값이 자연스럽게 이어질 수 있다.
      - 정규화된 값에서 가장 작은 지수는 $E = 1 - bias$이므로,
      비정규화된 값에서 $E = 1 - bias$를 사용하면
      비정규화된 값들이 정규화된 값들 바로 아래를 채우게 된다.
      - 즉, 비정규화된 값에서 $E = 1 - bias$를 사용하는 이유는
      비정규화된 값으로 표현되는 $0 \leq M < 1$의 범위의 수들과
      가장 작은 정규화된 수($1 \leq M < 2$)가 끊기지 않고 자연스럽게 이어지도록 하기 위한 것이다.
      - 만약 비정규화된 값에서 단순히 $E = 0 - bias$를 사용했다면
      비정규화된 값의 가장 큰 값과 정규화된 값의 가장 작은 값 사이에 표현의 공백이 생기게 된다.

## 3. Example Numbers - 예시 값들

### 정리

1. 정규화된 값의 계산 예시(책에는 없음)
    - 부호비트 1비트 | $exp$ 4비트 | $frac$ 3비트 형태
    - `[0 0010 010]` : 
      - $s$ = $0$ = 양수
      - $frac$ = $010$
      - $exp$ = $0010$ = $2$
      - $exp$가 전부 1이거나 전부 0이 아니므로 정규화된 값
      - $M$ = $1.frac$ = $1.010_{2}$
      - $E$ = $2 - bias$ = $2 - (2^{3} - 1)$ = $2 - 7$ = $-5$
      - $V$ = $(-1)^{0} \times 1.010_{2} \times 2^{-5}$
      - $V$ = $+1.010 \times 2^{-5}$
      - $V$ = $+0.00001010_{2}$
      - $V$ = $+0.00001010_{2} = \frac{1}{32} + \frac{1}{128} = \frac{5}{128} = 0.0390625_{10}$
      
2. 비정규화된 값의 계산 예시(책에는 없음)
    - 부호비트 1비트 | $exp$ 4비트 | $frac$ 3비트 형태
    - `1 0000 101` :
      - $s$ = $1$ = 음수
      - $frac$ = $101$
      - $exp$ = $0000$ = $0$
      - $exp$가 전부 0이므로 비정규화된 값
      - $M$ = $0.frac$ = $0.101_{2}$
      - $E$ = $1 - bias$ = $1 - (2^{3} - 1)$ = $1 - 7$ = $-6$
      - $V$ = $(-1)^{1} \times 0.101_{2} \times 2^{-6}$
      - $V$ = $-0.101 \times 2^{-6}$
      - $V$ = $-0.000000101_{2}$
      - $V$ = $-0.000000101_{2} = -(\frac{1}{128} + \frac{1}{512}) = -\frac{5}{512} = -0.009765625_{10}$
      
3. 특수한 값의 계산 예시(책에는 없음)
    - 부호비트 1비트 | $exp$ 4비트 | $frac$ 3비트 형태
    - `1 1111 000` :
      - $s$ = $1$ = 음수
      - $frac$ = 000
      - $exp$ = $1111$
      - $exp$가 전부 1이므로 특수한 값
      - $frac$이 전부 0이므로 무한대
      - $V = - \infty$
      
4. 정규화된 값의 최솟값과 비정규화된 값의 최댓값 비교(책에는 없음)
    - 부호비트 1비트 | $exp$ 3비트 | $frac$ 2비트 형태
    - 정규화된 값의 최솟값 : $exp = 001$ / $frac = 00$
    - $M = 1.00_{2}$ / $E = 1 - bias = 1 - (2^{2} - 1) = 1 - 3 = -2$
    - 즉, 해당 형태의 부동소수점 표현에서 정규화된 값의 최솟값은 
    $1.00_{2} \times 2^{-2} = 0.01_{2} = 2^{-2} = \frac{1}{4} = 0.25_{10}$이다.
    - 비정규화된 값의 최댓값 : $exp = 000$ / $frac = 11$
    - $M = 0.11_{2}$ / $E = 1 - bias = -2$
    - 즉, 해당 형태의 부동소수점 표현에서 비정규화된 값의 최댓값은
    $0.11_{2} \times 2^{-2} = 0.0011_{2} = 2^{-3} + 2^{-4} = \frac{3}{16} = 0.1875_{10}$이다.
    - 정규화된 값의 최솟값 : $\frac{4}{16}$ = $0.25_{10}$ / 비정규화된 값의 최댓값 : $\frac{3}{16}$ = $0.1875_{10}$
    - 비정규화된 값의 범위에는 0도 포함되어있음
    - 이로써, 비정규화된 값은 정규화된 값과 0 사이의 간극을 메운다는걸 알 수 있다.
    
5. 값의 증가에 따른 비트 패턴의 변화
    - IEEE 754에서는 $frac$부분의 비트가 전부 $1$이 된다면, 
    다음 수는 $frac$이 전부 0이 되고 $exp$부분이 $1$ 증가하는 식으로 
    값이 증가할 때마다 비트 패턴이 변한다.
    - 물론, 특수값 영역 전까지의 양수 유한 값의 범위이다.
    - 마치, $frac$이 낮은 자리수이고 $exp$가 그보다 높은 자리수인 것처럼 보인다.
    - 즉, 값의 증가에 따른 비트 패턴의 변화만 본다면,
    $frac$ 증가
    $\to$ $frac$이 가득참
    $\to$ $exp$ 증가
    $\to$ $frac$은 다시 $0$부터 시작
    이런 흐름으로 볼 수 있다.
    - 실제 $exp$ : 4비트 $frac$ : 3비트 형태로 예시를 들어보면,
     $exp$ $frac$
     `0000 000` 
     `0000 001`
     `0000 010` 
     `...` 
     `0000 111`
     `0001 000` 
     `0001 001` 
     `...` 
     `0001 111` 
     `0010 000` 
     `...`
     - 이렇게 $frac$이 가득차면 $exp$가 1 증가하는 것을 볼 수 있다.
     - 이건 마치, unsigned 정수의 증가와 비슷하게 비트 패턴이 변화하는 것으로 볼 수 있다.
     - 이건 우연이 아니고 IEEE 754 포맷이 일부러 이렇게 디자인 된 것이다.
     - 다만, 실제 의미는 unsigned 정수의 증가와 다르다.
     unsigned 정수에서 높은 자리수가 증가한다면 단순히 더 큰 자릿값이 더해지는 것 뿐이지만,
     부동소수점에서 $exp$가 증가하면 소수점 위치가 변하는 것이기 때문에,
     $exp$가 커질수록 표현 가능한 값들 사이의 간격도 함께 커진다.
     
6. $exp$ 증가에 따른 소수의 간격 증가
    - $exp$는 지수(소수점의 위치)를 나타내어 1 증가할 때마다 표현하는 값의 범위가 두 배씩 커지지만,
    $frac$은 한 $exp$ 구간에서 고정된 비트 수로 표현이 되기 때문에,
    $exp$ 구간이 늘어난다고해도 정해진 $frac$의 갯수가 동일하다.
    - 그렇기 때문에, 같은 개수의 $frac$ 패턴으로 두 배 더 큰 구간을 표현해야하기 때문에,
    $exp$가 커질 때 마다 각 $frac$에 의한 값의 간격은 두 배씩 벌어진다.
    - 예 : `001 00`과 `001 01` / `010 00`과 `010 01`의 차이를 보자.
     둘 다 $frac$이 1 증가했지만,
      - `001 00` / `001 01`은 $1.00_{2} \times 2^{-2}$ / $1.01 \times 2^{-2}$
      즉 $0.01_{2} = \frac{1}{4}$ / $0.0101 = \frac{5}{16}$, 
      $\frac{1}{16}$의 차이가 난다.
      - `010 00` / `010 01`은 $1.00_{2} \times 2^{-1}$ / $1.01 \times 2^{-1}$
      즉 $0.1_{2} = \frac{1}{2}$ / $0.101 = \frac{5}{8}$,
      $\frac{1}{8}$의 차이가 난다.
    - 이렇게, 비트열의 관점에서는 아랫자리 수에서 윗자리 수로 unsigned처럼 올림되는 것 같아 보이지만,
    $exp$가 지수를 나타내기에, 즉 소수점의 위치를 변화시키기에
    $exp$가 커질수록 표현 가능한 값들 사이의 간격도 함께 커지게 되는 것이다.
    
7. $frac$이 가득 찬 뒤 $exp$가 증가할 때 값이 자연스럽게 이어지는 이유
    - 한 $exp$ 구간의 마지막 값은 다음 $exp$ 구간의 시작값 바로 직전까지 도달하기 때문에,
    다음 $exp$ 구간으로 넘어갈 때 자연스럽게 값이 이어진다.
    - 예시 : `001 111`과  `010 000`으로 알아보자.
      - `001 111` 
        - $exp$ = $001$ = $1$
        - $frac$ = $111$
        - 정규화된 값이므로 $M = 1.111_{2}$
        - $E$ = $1 - bias$ = $1 - 3$ = $-2$
        - $V$ = $1.111_{2} \times 2^{-2}$ = $0.01111$ = $\frac{15}{32}$
      - `010 000`
        - $exp$ = $010$ = $2$
        - $frac$ = $000$
        - 정규화된 값이므로 $M = 1.000_{2}$
        - $E$ = $2 - bias$ = $2 - 3$ = $-1$
        - $V$ = $1.000_{2} \times 2^{-1}$ = $0.1$ = $\frac{1}{2}$ = $\frac{16}{32}$
      - 이렇게 $frac$이 전부 $1$로 찬 뒤 $exp$가 증가하고 $frac$이 다시 전부 $0$으로 되돌아가게 되어도,
      다음 $exp$ 구간은 그 바로 다음 값에서 시작하기 때문에,
      $frac$이 전부 $1$로 차서 올림이 발생해도
      문제가 되지 않는다.
