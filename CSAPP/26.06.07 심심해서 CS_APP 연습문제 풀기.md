# 연습문제나 풀자

## 2.31

```c
// 두 수를 더할 때 오버플로우가 발생하나 발생하지 않나 검사하는 함수..
// 버그 있음
int tadd_ok(int x, int y){
    int sum = x + y;
    return (sum-x == y) && (sum - y == x);
}
```

이게 왜 틀렸나 설명해보기..

Case 별로 나누어서 생각해보자..

결국 한 경우라도 오버플로우가 발생한 것을 제대로 판단 못한다면 해당 함수는 버그가 있고,
해당 경우를 찾아내면 왜 버그가 발생하는지 알 수 있겠지!

Case 1. x와 y를 더했을 때 오버플로우가 발생한다.

sum = x + y
(비트열 예시)
x == [0111]
y == [0011]
(당연하지만 오버플로우 발생) (가중치가 음수인 비트로 넘어가니까..)
sum == [1010]

sum - y = [1010] - [0011] = [1010] + [1101] = [0111] = x
sum - x = [1010] - [0111] = [1010] + [1001] = [0011] = y

이걸론 오버플로우 판별이 안되는데.. 어쨌든 이게 반례..

즉, 이 함수는 오버플로우가 발생했는데 참이 나오는 경우가 있음.
(모든 (x, y)에 대해서 참이 나오는지는 증명 못하겠다..)

### 증명해보자..

세 경우로 나눌 수 있을 듯.

- positive overflow
- negative overflow
- no overflow(정상 범위)

이 세 경우를 각각 증명해보자(경우에 의한 증명)

1. positive overflow
    - (전제)
      - x, y는 $w$비트의 2의보수 정수
      - $(x + y) > TMax_{w}$ (positive overflow 발생)
    - (추론)
      - $sum = (x + y) \bmod 2^{w} = x + y - 2^{w}$
      - $sum - y = ((x + y - 2^{w}) - y) \bmod 2^{w} = (x - 2^{w}) \bmod 2^{w}$
      - $sum - y = x$
    - (결론)
      - $\therefore$ positive overflow가 발생했을 때 sum - y = x이다.
2. negative overflow
    - (전제)
      - x, y는 $w$비트의 2의보수 정수
      - $(x + y) < TMin_{w}$ (negative overflow 발생)
    - (추론)
      - $sum = (x + y) \bmod 2^{w} = x + y + 2^{w}$
      - $sum - y = ((x + y + 2^{w}) - y) \bmod 2^{w} = (x + 2^{w}) \bmod 2^{w}$
      - $sum - y = x$
    - (결론)
      - $\therefore$ negative overflow가 발생했을 때 sum - y = x이다.
3. no overflow
    - (전제)
      - x, y는 $w$비트의 2의보수 정수
      - $TMin_{w} \leq (x + y) \leq TMax_{w}$
    - (추론)
      - $sum = (x + y) \bmod 2^{w} = x + y$
      - $sum - y = ((x + y) - y) \bmod 2^{w} = x \bmod 2^{w}$
      - $sum - y = x$
    - (결론)
      - $\therefore$ 오버플로우가 발생하지 않았을 때 sum - y = x이다.

모든 경우에서 sum - y = x이니, 해당 코드는 항상 참이다.

### 좀 고쳐보자..

- 2의보수 정수의 덧셈에서 오버플로우가 발생한걸 판단하려면..
  - 양수 + 양수 = 음수
  - 음수 + 음수 = 양수
  - 이것만 보면 됨..

```c
int tadd_ok(int x, int y) {
    int sum = x + y;
    bool neg_overflow = (x < 0) && (y < 0) && (sum >= 0);
    bool pos_overflow = (x >= 0) && (y >= 0) && (sum < 0);

    return !(neg_overflow || pos_overflow);
}
```

### 2.35

```c
// 이건 왜 맞을까? 이것도 2.31처럼 역산하는건데?
int tmult_ok(int x, int y) {
    int p = x * y;

    return !x || p / x == y;
}
```

(전제)

1. 해당 코드에서 p가 오버플로우가 발생하지 않았다면 p/x == y가 나와야 하고, 그 반대도 성립한다.(필요충분조건)
2. 오버플로우가 발생했다면 p의 값은 x * y + t2^{w}가 된다. 즉, 오버플로우가 발생하지 않았다면 t == 0이어야 한다.
3. p / x는 p = x * q + r으로 표현될 수 있다. r은 나머지이므로, |r| < |x|를 만족시킨다.

(추론)

- 오버플로우가 발생하지 않았다면 p / x == y이므로 q == y라고 할 수 있다.
- p / x == y는 p = x * y + r으로 표현할 수 있다. q == y라면.
- 결국, x * y + t2^{w} = x * y + r을 전개하면 t2^{w} = r이 나온다.
- 하지만 |r| < |x| < 2^{w}이므로 t2^{w} = r이라면 t가 0이어야 말이 된다. 

(결론)
$\therefore$ 해당 코드에서 p가 오버플로우가 발생하지 않았다면(t == 0) p / x == y이다.
