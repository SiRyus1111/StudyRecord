## 1. Integer Data Types - 정수 자료형

### 요약

1. C가 지원하는 정수 자료형들
- 대표적으로
- char, short, long 등등..

2. 정수형의 특징 1 - long은 실행하는 환경에 영향을 크게 받는다.
- 예를 들어,
- 32비트 장치에서는 4바이트, 값의 범위(-2147483648 ~ 2147483647)
- 64비트 장치에서는 8바이트, 값의 범위(-922경 ~ +922경)(너무 길어서 생략함)
- 물론 이건 절대적인건 아니다. OS에 따라, 컴파일러에 따라 달라질 수 있다.
- 실제로 윈도우 64비트에서는 long이 4바이트인 경우가 흔하다. 이렇게 예외도 있다.
- 어쨌든 실행하는 환경에 영향을 크게 받는다. 아예 해당 자료형의 바이트 수가 변한다.

3. 정수형의 특징 2 - 대부분의 signed 자료형들은 음수 범위가 양수 범위보다 1 크다.
- \[signed] char = -128 ~ 127
- short = -32768 ~ 32767
- int = -2147483648 ~ 2147483647
- 왜 이렇게 음수가 표현되는지 좀 이따 알아본다고 한다.

4. C언어가 따로 보장하는 각 정수 자료형들의 값의 범위가 있다.
- \[signed] char = -127 ~ 127
- int = -32767 ~ 32767
- long = -2147483647 ~ 2147483647
- C언어가 보장하는 int형의 범위가 16비트 범위인건 이전의 16비트 장치의 영향이다.
- 그리고 C언어가 따로 보장하는 각 정수 자료형들의 값의 범위는 실제 우리가 쓰는 것처럼
음수 범위가 양수 범위보다 1 크지 않고 대칭을 이루고 있다.

### 느낀 점 & 배운 점

뭔가 본격적인 설명에 들어가기 앞서
간단하게 기본적인 정보들을 설명해주는 느낌이 드는 파트였다.

일단 내가 프로그래밍을 조금은 해본 사람인지라,
자료형의 종류는 대부분 알고 있었지만,
long 형의 값의 범위가 실행하는 장치에 영향을 받는다는 사실은
이 책을 읽으면서 처음으로 알게된 것 같다.

그리고 저렇게 음수 범위가 양수 범위보다 1 큰 것에 대해
항상 의문을 가지고 있었는데,
이번에 확실히 알아갈 수 있다는 점 때문에 앞으로의 파트들이 기대된다.

C언어가 따로 보장하는 각 정수 자료형들의 값의 범위가 있다는 것도 처음 알았다.
실제로 코딩할 때 프로그램이 실행되는 장치에 상관없이 
각 자료형이 값을 표현가능한 범위의 하한선이라고 생각하면 될 것 같다.
일종의 세이프라인?
저 범위 이내라면 C언어로 코딩할 때 어떤 장치든 문제가 생기지 않는다고 이해했다.
말그대로 각 정수 자료형의 표현 가능한 수의 범위의 하한선같은 느낌이었다.

그리고 특히 int형의 C언어가 보장하는 값의 범위가
16비트 범위인게 신기했다.
그냥 16비트 장치를 사용했을 때의 잔재인데,
그것까지 고려하는 C언어 표준이 참 신기했다.

그리고 분명 실제 signed 자료형들은 음수 범위가 양수 범위보다 1이 큰데,
왜 C언어가 보장하는 정수 자료형들의 값의 범위에서는
음수의 범위와 양수의 범위가 대칭을 이루는지도 궁금하다.

## 2. Unsigned Encodings - 부호없는 정수의 인코딩

### 요약

1. $B2U$ - Binary to Unsigned Integer(2진수 비트열을 부호없는 정수로 변환하는 함수)

$$B2U_w(\vec{x}) \doteq \sum_{i=0}^{w-1} x_i 2^i
$$

- 여기서 $\doteq$는 왼쪽($B2U_w(\vec{x})$)이 오른쪽($\sum_{i=0}^{w-1} x_i 2^i$)으로 정의된다는 의미의 기호이다.
- 이 수식이 어려워보이지만, 사실 정말 쉬운 수식이다.
- $B2U_w$는 $w$비트의 비트열 $\vec{x}$를 부호없는 정수로 해석하는 **함수**라고 보면 된다.
- 그리고 이 수식의 의미는, 
각 비트($x_i$, $x_0$ = 맨 오른쪽 비트)에 그 자릿수의 가중치($2^i$)를 곱한 뒤 전부 더하라($\sum$)는 뜻이다.
- 정말정말 쉽게 설명하자면, 그냥 우리가 평소에 2진수를 숫자로 변환할 때와 사실상 다를 바가 없다.

2. $B2U$의 특성 - 비트의 수($w$)가 고정되어있을 때, 표현할 수 있는 값의 범위는 어떻게 될까?
- 가장 큰 부호없는 정수는 각 비트 $x_i$가 모두 1일 때이다.
- 즉,
$B2U_w(\vec{x}) \doteq \sum_{i=0}^{w-1} 1 \times 2^i$
인 경우가 가장 큰 값($UMax_w$)이다.
- 따라서, $w$비트의 부호없는 정수가 표현할 수 있는 최댓값($UMax_w$)은 
$UMax_w = 2^w - 1$
이다.
- 즉, $w$비트의 부호없는 정수(Unsigned Integer)는 $0$ ~ $2^w - 1$까지의 값을 표현할 수 있다.

3. $B2U$의 특성 - 일대일 대응(bijection)
- 고정된 $w$비트의 부호없는 정수 값에서 표현할 수 있는 모든 숫자들은 전부 
$0$ ~ $2^w - 1$ 범위의 중복되지 않는, 독립된 값을 가진다.
- 예를 들어서, 부호없는 10진수 정수 11은 
오직 1011이라는 4비트의 2진수로만 표현될 수 있다.
1011이 아닌 다른 2진수는 10진수 정수 11이라는 값을 나타내지 못하고,
11이 아닌 다른 10진수는 2진수 1011이라는 값을 나타내지 못한다.
- 이것을 책에서는 Uniqueness of unsigned encoding(부호없는 정수 인코딩의 고유성)
이라고 한다.
- 즉, 일대일 대응(bijection) - $f(x) = y,\; f^{-1}(y) = x$
이렇게 하나의 $x$값으로는 하나의 $y$값만 대응될 수 있고, 
하나의 $y$값도 하나의 $x$값만 대응될 수 있는
특성을 가진다.
- 뭔가 간단하고 당연해보이지만, 
컴퓨터 입장에서는 하나의 비트열을 하나의 값으로 일관되게 해석해야하므로,
이 특성이 상당히 중요하다.

### 느낀 점 & 배운 점

직접 한번 $B2U_4$로 예시를 들어보았다.

- 예시 ) 
  - 만약 $B2U_4([0101])$ 이렇다면,
  - $x_0 = 1$(맨 오른쪽 비트), $2^0 = 1$이므로, $1 × 1 = 1$
  - $x_1 = 0$, $2^1 = 2$이므로, $0 × 2 = 0$, 앞의 값들과 계속 더해가니 $1 + 0 = 1$
  - $x_2 = 1$, $2^2 = 4$이므로, $1 × 4 = 4$, 앞의 값들과 계속 더해가니 $1 + 0 + 4 = 5$
  - $x_3 = 0$, $2^3 = 8$이므로, $0 × 8 = 0$, 앞의 값들과 계속 더해가니 $1 + 0 + 4 + 0 = 5$
  - 따라서 이진수 0101을 부호없는 정수(Unsigned Integer)로 변환($B2U$)하면 5라는 것이다.
- 수식으로 한 줄 요약
  - $B2U_4([0101]) = 0 × 2^3 + 1 × 2^2 + 0 × 2^1 + 1 × 2^0 = 0 + 4+ 0 + 1 = 5$
  - 이건 왼쪽 자리수부터 표기함.
  
이렇게 예시를 들어보았다.

이렇게 직접 설명하듯이 $B2U$를 해보니 확실히 $B2U$가 뭔지 체감이 되는 것 같다.
그냥 평소 하던 대로 평범하게 2진수를 10진수로 바꾸면 되는 것 같았다.

그리고  $B2U$의 특성을 제대로 살펴보니,
평소에는 당연하다고 생각했던 것들이 논리적으로 이해가 된 느낌이다.
- 고정된 $w$비트의 부호없는 정수(unsigned integer)형이 표현 가능한 수의 범위라던지
- 고정된 $w$비트의 2진수와 부호없는 정수가 일대일 대응이라는게 중요한 이유라던지..

원래 이런 것들은 '당연히 그렇지'라고 생각하고 있었지만,
좀 더 논리적인 설명을 듣고 이해를 해보니 확실히 체감이 된 것 같다.

## 3. Two's Complement Encoding - 2의 보수의 인코딩

### 요약

1. 오늘날 대부분의 컴퓨터는 2의 보수 형태로 음수를 표현한다.
- 2의 보수는 맨 앞 비트(최상위 비트, 맨 왼쪽 비트)가 음의 가중치를 가진다.
- 예를 들어,
4비트 2의 보수 표현에서 1011은,
- $1011 = 1 \times -2^3 + 0 \times 2^2 + 1 \times 2^1 + 1 \times 2^0 = -8 + 0 + 2 + 1 = -5$
- 보다시피, 맨 앞 비트(최상위 비트)의 가중치는 unsigned 형태였다면 $2^3$이었겠지만,
2의 보수이므로 최상위 비트의 가중치가 $-2^3$, 음수가 된다.
- 즉, 1011은 4비트 2의 보수 정수로 해석하면 **-5**이다.


2. $B2T$ - Binary to Two's-Complement(2진수 비트열을 2의 보수로 변환하는 함수)

$$B2T_w(\vec{x}) \doteq -x_{w-1} 2^{w-1} + \sum_{i=0}^{w-2} x_i 2^i
$$ 

- 이 수식도 뭔가 어려워보이지만, 사실 이전의 $B2U$에서 아주 조금 변형된 수식이다.
- $B2T_w$는 $w$비트의 비트열 $\vec{x}$를 $+$/$-$ 부호가 있는 정수로 해석하는 **함수**라고 보면 된다.
- 이 수식의 의미는, 
최상위 비트(맨 앞 비트, $x_{w-1}$)는 그 자릿수의 음수의 가중치($-2^{w-1}$)를 곱하고, 
최상위 비트를 제외한 각 비트($x_i$, $x_0$ = 맨 오른쪽 비트)에 그 자릿수의 가중치($2^i$)를 곱한 후 
전부 더하라($\sum$)는 뜻이다.
- 정말 쉽게 설명하자면, 
앞의 부호없는 정수로 변환하는 함수($B2U$)에서 
**최상위 비트의 가중치**만 **음수**로 생각해서 계산하는 것이다.

3. $B2T$의 특성 - 비트의 수($w$)가 고정되어있을 때, 2의 보수로 표현할 수 있는 값의 범위는 
어떻게 될까?
- 이것도 $B2U$와 상당히 비슷하다.
- 일단, 가장 작은 2의 보수($TMin$)는 최상위 비트(부호 비트, $x_{w-1}$)가 1이고, 
모든 나머지 비트($x_i$, 단 $x_{w-1}$은 제외)가 0인 경우이다.
- 즉,
$B2T_w(\vec{x}) \doteq 1 \times -(2^{w-1}) + \sum_{i=0}^{w-2} 0 \times 2^i$ 
인 경우가 가장 작은 2의 보수 값($TMin_w$)이다.
- 따라서, $w$비트의 2의 보수가 표현할 수 있는 최솟값($TMin_w$)은 
$$TMin_w = 1 \times -(2^{w-1}) + \sum_{i=0}^{w-2} 0 \times 2^i = 1 \times -(2^{w-1}) + 0 
$$
$$
= -(2^{w-1})
$$
이다.
- 그리고, 가장 큰 2의 보수($TMax$)는 최상위 비트(부호 비트, $x_{w-1}$)가 0이고,
모든 나머지 비트($x_i$, 단 $x_{w-1}$은 제외)가 1인 경우이다.
- 즉, 
$B2T_w(\vec{x}) \doteq 0 \times -(2^{w-1}) + \sum_{i=0}^{w-2} 1 \times 2^i$
인 경우가 가장 큰 2의 보수 값($TMax_w$)이다.
- 따라서, $w$비트의 2의 보수가 표현할 수 있는 최댓값($TMax_w$)은
$$TMax_w = 0 \times -(2^{w-1}) + \sum_{i=0}^{w-2} 1 \times 2^i = 0 + \sum_{i=0}^{w-2} 2^i 
$$
$$= 2^{w-1}-1
$$
이다.
- 이렇게 $w$비트의 2의 보수가 표현할 수 있는 최솟값($TMin_w$)과 최댓값($TMax_w$)을 알았으니,
$w$비트의 2의 보수(Two's-Complement)는 $-(2^{w-1})$ ~ $2^{w-1}-1$까지 표현할 수 있다.
- 근데 이렇다면, 양수 쪽보다 음수 쪽이 1만큼 더 큰 범위를 가지게 된다.
- 내가 앞에서 signed 자료형의 음수 범위가 왜 양수보다 1 큰지 궁금했었는데,
이유가 바로 이것이다.

4. $B2T$의 특성 - 일대일 대응(bijection)
- $B2T$도 $B2U$처럼 일대일 대응이다.
- 하나의 고정된 $w$비트의 2의 보수 표현에는
오직 하나의 10진수 정수가 대응된다.
그 반대도 성립한다.
($f(x) = y,\;f^{-1}(y) = x$)

5. 다양한 자료형에서의 $UMax_w$, $TMax_w$, $TMin_w$가 중요한 이유
- 앞에서 설명했던 수식들의 원리만 알아도,
몇 비트($w$)의 자료형이고, 
signed / unsigned 중 어느 형태로 데이터를 표현하는지에 관한 정보만 있으면
$UMax_w$ / $TMax_w$ / $TMin_w$를 전부 구할 수 있다.
- 그리고 각 자료형 크기의 $UMax$ / $TMax_w$ / $TMin_w$은 어느정도 잘 기억해두는게 좋다.
- 각 자료형으로 표현할 수 있는 수의 범위를 확실히 알아야,
2의 보수의 범위와 비트 표현때문에 생기는 다양한 문제를 예방하고, 대처할 수 있기 때문이다.
  - $TMin_w$의 값은 해당 고정 크기 자료형에서 그에 반대되는 양수의 값이 없기 때문에, 
  2의 보수의 산술 연산을 할 때의 미묘한(subtle) 프로그램 오류가 발생할 수 있다.
  결국 0이 양수 표현에 포함되기 때문에 발생하는 문제이다.
  - $UMax_w$는 $TMax_w$의 두 배 이상($UMax_w = 2 \times TMax_w + 1$)이기 때문에, 
  같은 바이트를 사용하는 자료형이더라도 
  2의 보수 표기법에서 음수를 나타내는 표기들이 unsigned 표기에서는 양수 값이 된다.
  실제로 8비트 2의 보수 표기법의 $-1$인 `0b11111111`은 
  unsigned로 표기하면 $UMax_{8} = 255$과 다르지 않은 것 처럼..
  
6. 추가 정보들
- C 표준은 signed 자료형에서 반드시 2의 보수 표기법을 사용하라고 강제하지는 않는다.
하지만 오늘날 거의 모든 시스템은 2의 보수 표기법을 사용한다.
- 이식성이 매우 높은 코드를 짜는 프로그래머는 프로그램이 어떤 장치에서 돌아갈지 모르기 때문에, 
C 표준이 보장하는 최소 범위를 넘지 않게 코드를 짜야한다.
- 실제로 $TMax_w$ / $TMin$ 같은 값들을 사용하는 경우에는 바로 하드코딩으로 그 값들을 넣지 말고,
C의 <limits.h>에 정의된 INT_MAX / INT_MIN / UINT_MAX 같은 상수를 쓰는게 정석이다.
