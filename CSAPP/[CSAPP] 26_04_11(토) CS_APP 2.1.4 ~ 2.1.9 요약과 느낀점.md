## 4. Bit-Level Operations in C - C언어에서의 비트 연산

### 요약

1. C언어의 비트 연산자들.
- NOT - ~
- OR - |
- AND - &
- XOR - ^
- 등등..

2. XOR 연산의 성질 - (A XOR A = 0)(A ^ A = 0)
- 두 피연산자의 각 비트가 서로 다를 때 1을 반환하므로,
같은 두 피연산자의 각 비트가 전부 같다면 전부 0을 반환한다.
- 이 성질을 이용해서, 두 변수의 값을 swap할 때, 굳이 temp를 쓸 필요가 없게 하는 등
다양하게 응용할 수 있다. 
물론 이렇게 temp없이 swap하는건 코드 가독성때문에 실무에선 거의 안쓴다.
- RAID 5 / 6의 패리티 비트?

3. 비트 연산의 사용법 - 비트 마스킹
- 각 연산으로
  - 특정 바이트만 남긴다던지(AND)
  - 특정 바이트 빼고 반전한다던지(NOT, XOR)
  - 특정 바이트만 변경한다던지(OR)
- 다양한 형태의 비트 마스킹을 할 수 있다.

4. 비트 연산으로 다양한 연산을 만들 수 있다.
- OR과 AND NOT을 엮어서 XOR 연산을 만든다던지 - ((B AND NOT A) OR (A AND NOT B)) = XOR
- 이런 식으로 기본적인 비트 연산들로 다양한 연산을 만들 수 있다.

### 느낀 점 & 배운 점

기존에 비트 연산과 비트 마스킹을 단순히 이론으로만 알았을 때와 비교해서 상당히 많이 배운 것 같다.
기존에는, 
> 서브넷 마스크 - 비트 AND 연산을 해서 네트워크 주소와 호스트 주소를 구분

> RAID 5 / 6의 패리티 비트 - 비트 XOR 연산의 특성인 A ^ A = 0을 이용해서
특정 디스크(A)가 고장난 경우 패리티 비트 = (A ^ B = P(패리티 비트))를 이용해서
B ^ P = A로 데이터를 복구할 수 있는 비트


$$
A \oplus B = P
$$

$$
A가\,유실될\,시,
$$
$$
P \oplus B = (A \oplus B) \oplus B
$$
$$
B \oplus B = 0\,이므로
$$
$$
A \oplus 0 = A
$$


이 정도로 이론적으로만 알고있었지만,

직접 XOR 연산을 각각의 변수를 a와 b로 놓고 계산해보는 실습을 해보고,

- 제시된 바이트의 특정 부분만 제외하고 전부 0으로 만들거나
- 특정 부분을 반전하거나
- 특정 부분만 바꾸거나하는
코드를 직접 짜보면서,
실제 이러한 비트 연산이 어떻게 일어날 수 있는지 몸소 깨닫게 된 것 같다.

실습 코드 첨부
```cpp
// C의 비트 연산 연습문제

// 1. 하위 1바이트만 그대로 유지하고 나머지 전부 0으로 바꾸기

// 2. 하위 1바이트만 그대로 유지하고 나머지 전부 뒤집기

// 3. 하위 1바이트만 모든 비트 1으로 바꾸기(하위 1바이트만 ff로 바꾸기)

#include <stdio.h>

int main(){
    int x = 0x87654321;

    // 1. 하위 1바이트만 그대로 유지하고 나머지 전부 0으로 바꾸기 = 0x00000021
    int mask1 = 0x000000ff;
    int res1 = x & mask1;
    printf("%x\n", res1);

    // 2. 하위 1바이트만 그대로 유지하고 나머지 전부 뒤집기 = 0x789abc21
    int mask2 = 0xffffff00;
    int res2 = x ^ mask2;
    printf("%x\n", res2);

    // 3. 하위 1바이트만 모든 비트 1으로 바꾸기(하위 1바이트만 ff로 바꾸기) = 876543ff
    int mask3 = 0x000000ff;
    int res3 = x | mask3;
    printf("%x\n", res3);

    return 0;
}
```

출력
```

789abc21
876543ff
```



## 5. Logical Operations in C - C언어에서의 논리 연산

### 요약

1. C언어의 논리 연산자들.
- NOT - !
- OR - ||
- AND - &&
- 0을 FALSE, 0 제외 모든 수는 TRUE로 취급한다.
- 결과는 0 / 1로만 나온다.

2. 비슷해보이는 비트 연산과의 차이
- 쉽게 보자면, 
- 비트 연산 = 불 대수(0과 1, 숫자에 대한 연산) - 결과로 연산된 비트가 숫자로 나옴
- 논리 연산 = 명제(참 / 거짓에 대한 연산) - 숫자지만, 참(1)과 거짓(0)을 의미하는 값만 나옴
- 논리 연산은 비트 연산과 다르게 첫 번째 피연산자만으로도 논리값(TRUE(1) / FALSE(0))이 결정되는 경우,
두 번째 피연산자를 딱히 평가하지(evaluate) 않는다.

### 느낀 점 & 배운 점

일단, 나는 기존에는 논리 연산을 그저 
> true와 false로 결과가 나오는 연산

이라고만 생각하고 있었는데,
더 확실히 알게된 것 같아.

논리 연산에서 어떤 값들이 TRUE, 어떤 값들이 FALSE로 인식되는지.

논리 연산과 비트 연산의 결정적인 차이점인 
논리 연산은 첫 번째 피연산자만으로 논리값이 결정되면,
예를 들어 논리 AND(&&) 연산을 하는데 이미 첫 번째 피연산자가 0이라면..
이럴 때 두 번째 피연산자를 딱히 평가하지 않는다는 점.

 >비트 연산 = 숫자
논리 연산 = 명제

이렇게 직관적으로 와닫는 비유..

논리 연산에 대해 좀 더 잘 알게된 것 같아.

이렇게 연습 문제를 풀어보며, 
==연산자 같은 연산자도 비트 연산과 논리 연산으로 구현할 수 있다는걸 알았어.

```cpp
// C언어의 논리연산 연습문제
// 오직 비트 연산과 논리 연산만으로 x == y를 판별해서 참이면 1, 거짓이라면 0을 반환하는 함수 작성

// A ^ A = 0

#include <stdio.h>

int is_equal(int x, int y){
    int XOR_val = !(x ^ y); // x와 y가 같다면 XOR는 0을 반환하고, 그걸 논리 NOT 연산자로 뒤집으면 된다..
    return XOR_val;
}

int main(){
    int x_arr[7] = {1, 2, 3, 4, 5, 6, 7};
    int y_arr[7] = {1, 3, 4, 4, 5, 7, 7};

    for (int i = 0; i < 7; i++){
        printf("%d\n", is_equal(x_arr[i], y_arr[i]));
    }

    return 0;
}
```

## 6. Shift Operation in C - C언어에서의 시프트 연산

### 요약

1. C언어의 시프트 연산자들
- 왼쪽 시프트(왼쪽으로 k비트만큼 비트 밀기) - x `<<` k
- 오른쪽 시프트(오른쪽으로 k비트만큼 비트 밀기) - x `>>` k

2. 왼쪽 시프트 연산
- `x << k` 형식으로 쓴다.
- x를 비트단위로 k만큼 왼쪽으로 민다.
- 비트를 밀고 오른쪽에 남은 빈칸들에는 0이 채워진다.
- 자료형의 최대 크기(int 기준 32비트(4바이트), char 기준 8비트(1바이트) 등) 이상으로 
비트를 밀 수 없다.

3. 오른쪽 시프트 연산
- `x >> k` 형식으로 쓴다.
- x를 비트단위로 k만큼 오른쪽으로 민다.
- 여기서 두 가지 오른쪽 시프트 연산이 있다.
  - 논리(Logical) 시프트 : 비트를 밀고 왼쪽에 남은 빈칸들에는 오직 0만 채워진다.
  - 산술(Arithmetic) 시프트 : 비트를 밀고 왼쪽에 남은 빈칸들에 
  기존의 최상위 비트(맨 왼쪽 비트)가 채워진다.
  최상위 비트(맨 왼쪽 비트)란, 8비트(1바이트) 자료형 기준,
  1000 1101이라면 1이고, 0011 0111이라면 0이다.
  그냥 전체 비트에서 맨 왼쪽 비트라고 생각하면 편하다.
  
- 이렇게 두 가지 방식의 오른쪽 시프트 연산을 사용하는 이유는,
signed(부호 있는) 정수형 때문이다.
- signed 정수형(integer)은 맨 왼쪽 비트를 부호를 표시하는데 쓰는데, 이걸 부호 비트라고 한다.
- 오른쪽 시프트할 때 0을 채워버리면 
부호 비트가 1일 때, 그러니까 음수를 나타내고 있을 때는
아예 값의 의미가 달라진다.
- 그래서 부호 비트를 맞춰서 빈 비트들을 채워넣어서, 
값의 의미를 다르게 하지 않고 시프트를 하기 위해서 산술 시프트와 논리 시프트가 구분되어있다.
- 물론 unsigned(부호 없는, 음수 불가능) 정수형은 논리 시프트를 사용한다. 그래도 문제가 없다.
- 그리고 이 시프트도 물론 자료형의 최대 크기 이상으로 
비트를 밀 수 없다.
